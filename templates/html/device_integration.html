Edge Integration

Overview

The Edge device integration enables direct integration from the physical edge device to a local endpoint in the device network.

The Edge integration does not require connectivity to the innoVi core (in the cloud or on-prem data center) but it includes only raw data (without enrichment)

There are only two data integration available with the edge integration:

  • Events integration - The device will push raw event data on any real-time analytics event detection (not including anomaly detection).
  • Metadata integration - The device will stream the metadata to the integration endpoint.

To enable device integration type and endpoint, the user must first define integration target:

  1. In the innoVi portal, go to Settings | Integration targets tab.
  2. Add new integration target, only WebHook (HTTP/S) or WebSocket (WS/S) protocols are supported for Edge integration.
  3. Set protocol definitions (local endpoint, HTTP verb and HTTP headers)
  4. In the Device page, go to Integrations tab
  5. Enable the integration type (Events/Metadata>) and select the integration target

Best Practices

  1. Events integration - It is recommended to use WebHook (HTTP/S) protocol due to the payload size (event image and event history)
  2. Metadata integration - It is recommended to use WebSocket (WS/S) protocol due to the metadata throughput

Data Types
