Web Sockets

Overview

The innoVi Core API exposes web socket endpoints for real-time push notification to client applications.

To use the API, the following HTTP headers are required:

  • X-API-KEY - Any consumer MUST include API key provided by Agent Vi. API Keys are used to identify the services consumer (application/service) type and grant access to relevant resources only.
  • X-ACCESS-TOKEN - Security context to identify the Web Socket consumer and the account info.

These parameters are required for the initial web socket connection. After protocol upgrade from the HTTP to Web Socket, the security token (which encapsulates user and account info) should be provided in the token query parameter.
Since the token is provided in the URL, the token should be URL encoded in order to ensure the URL is valid.
